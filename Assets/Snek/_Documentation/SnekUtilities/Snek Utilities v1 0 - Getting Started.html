Snek Utilities v1.0 - Getting Started

General Overview

What this package does:

  • Provides a cleaner, less error-prone and a bit more advanced MonoBehaviour workflow
    - MonoBehaviour vs SnekMonoBehaviour
  • Allows a bit more modern visual style for default inspectors without any code
    - Modern Customizable Default Inspectors
  • Adds the option to make a ScriptableObject persistent, causing it to auto-generate if missing and allowing it to be very easily referenced for in-editor workflow - Auto-Generated Scriptable Objects
    (Very useful for creating editor tools)
  • Provides advanced custom inspector classes for MonoBehaviour and ScriptableObject
    - Advanced Custom Inspector
  • Simplifies coding for custom inspectors and custom editor windows with reusable classes for common GUI elements - Advanced Property Drawers
    (This prevent boilerplate code and allows you to focus on the actual logic instead of spending a lot of time GUI drawing)
  • Introduces a DropBox GUI element and an advanced version of ReorderableList - Drop Box,
    Advanced ReorderableList
  • A few additional small editor utilities

This document contains only a brief introduction to the package content.
For more details and notes, you can check the Public API Documentation.

This package was originally intended as my own base for developing tools so it will be updated with new things every now and then.

MonoBehaviour vs SnekMonoBehaviour

MonoBehaviour

Default GameObject flow.

  • Awake()
  • Start()

SnekMonoBehaviour

Contains initialization callbacks while still allowing the usage of Start() and Awake() through overrides.

  • Initialize()
  • IsInitializedInStart() - you choose whether Initialize() is called in Start() or Awake()
  • Validate()
  • OnFailValidation()
  • OnInitializationSuccess()
  • FailValidation(string message) - your message so you know exactly what caused the validation to fail, eg. if some required reference is null or a value is incorrect.
Modern Customizable Default Inspectors
How to apply

Simple add the [UseSnekInspector] attribute on a SnekMonoBehaviour or a SnekScriptableObject class

Additional options

You can disable the entire inspector interaction by using the [UseSnekInspector(true)] attribute overload or disable specific simple serialized variables by adding [SnekInspectOnly] attribute to them instead.

Color scheme modifications

If you don’t like the default colors, you can use your own. Just change the values in SnekInspectorGUILayoutSettings asset.

Auto-Generated Scriptable Objects

To create an auto-generated ScriptableObject, simply add the [SnekAutoGenerateAsset] attribute to a SnekScriptableObject.

There are two overloads to choose from:

  1. [SnekAutoGenerateAsset(string folderPath, string fileName)]
  1. [SnekAutoGenerateAsset(string targetAssetPath)] - make sure to add “.asset” at the end for this one

Such auto-generated assets can be easily referenced through a singleton-like pattern for in-editor workflow which is very useful for creating editor tools for which you need some data stored in the project.

To get a reference to an auto-generated asset, use ScriptableObjectManager.GetAsset<T>();

Advanced Custom Inspector
How to

There are two base classes when creating a custom inspector:

  1. SnekMonobehaviourInspectorCustom<T>
  1. SnekScriptableObjectInspectorCustom<T>

Both of them come with additional callbacks which you can override, also updating the property values of the inspected SerializedObject is done internally so you dont need to write serializedObject.Update(); and serializedObject.ApplyModifiedProperties(); every time.

To use either of those base inspectors, you simply inherit from the correct one based on the inspected object type instead of inheriting from Editor class.

Modular base property drawing

You can mix the base inspector with the custom inspector by hiding properties from the base inspector instead of forcing one or the other.

You can show/hide properties in the base inspector logic by using HideProperty() to hide a property and ShowProperty() to show a previously hidden property again.
You can also simultaneously get a reference to a SerializedProperty of the inspected object and hide it by using GetProperty(string propertyName, bool hideFromBaseProperties) overload.

Drop Box

Usually, when you have a regular List<T> or Array in the inspector, you can do the Drag & Drop to add new elements, but you need to aim at the property label which can be quite annoying, especially on bigger screen resolutions since everything is smaller.
SnekDropBox GUI element takes away that problem by allowing you to choose where you need to drop things, making it easier to work with.

SnekDropBox can be drawn over a provided Rect, or as a standalone GUILayout element:

  • Rect approach example : _dropBox.Draw(Rect rect)
  • Standalone approach example : _dropBox.DrawStandalone(string label)

Advanced ReorderableList

SnekReorderableList is a more advanced version of ReorderableList class which was originally intended to be used as the base class for your custom list classes, but can be used as-is to draw a List<T> or Array in a nicer style easily.

Using SnekReorderableList as the base class instead, makes drawing custom lists in inspectors/windows more intuitive, provides more options through new additional callbacks, and allows custom GUI styles to be used for most parts.
For more details on this approach, check the Public API Documentation.
